## ◼ FEATURED SYSTEM — ESchoolPay

Not just an application.
A **transactional ecosystem** connecting parents, schools, and payment infrastructure into a single reactive flow.

```
  [ Parent App ]  ──Flutter+GetX──▶  [ API Gateway ]  ──Laravel 12──▶
  [ Business Core: Enrollments · Payments · Children · Accounts ]   ──▶
  [ MySQL Data Layer ]  ──▶  [ Payment Orchestration: Mobile Money · BEAC ]
```

| Layer | Technology | Level |
|-------|-----------|-------|
| Interface | Flutter + GetX | 🔵 Advanced |
| Core API | Laravel 12 | 🔵 Advanced |
| Auth | OAuth 2.0 / Passport | 🔵 Advanced |
| Admin Panel | Filament v3 | 🔴 Intermediate |
| Data | MySQL | 🔵 Advanced |
| Infra | Docker (in progress) | 🟡 Learning |

## ◼ AFRICAN CONTEXT ENGINE

```
Constraints define innovation.
```

```
  Low bandwidth         →  🔵 Lightweight calls, local caching
  Mobile-first usage    →  🔵 Flutter offline-ready guards
  Local payments        →  🔴 BEAC / ClikPay / CinetPay aware
  Variable connectivity →  🔴 Retry logic, queue-based flows
  DevOps on-prem        →  🟡 Docker + Ansible automation
```

> Constraint is not a limitation. It is the design driver.
